# data-migration-plan

_Authoring guidance for the `data-migration-plan` document type — audience, depth, and rules._

- Audience: the operator on migration day and the reviewer before it.
- Filename: `docs/data-migration-plan-<subject>.html`.
- Depth: full.
- Rules: the mapping table is exhaustive (unmapped fields listed as
  consciously dropped); validation is quantitative (counts, checksums) —
  "looks fine" is not validation; the point of no return is marked in the
  procedure; rehearse on a copy and record the rehearsal result.
