# maintenance-plan

_Authoring guidance for the `maintenance-plan` document type — audience, depth, and rules._

- Audience: the maintainer and the auditor of maintenance.
- Filename: `docs/maintenance-plan-<equipment>.html`.
- Depth: full.
- Rules: intervals justified (manufacturer, regulation, or failure history);
  each task's steps carry measurable acceptance criteria; the schedule table
  is kept current (last/next dates) — an outdated plan is worse than none,
  it certifies false confidence.
